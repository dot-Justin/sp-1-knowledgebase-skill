## Narrative

**TimK posts demo video:** A video demonstrating custom firmware playing user-provided stems on a real TE-SP-1. Playback, stem switching, tempo sync - all working. The thread treats this as proof-of-concept complete. The device is now a general-purpose stem player for any audio, not just Ye's content.

**Timing bytes fully decoded:** The 16-bit timing counter is completely understood:
- Range: **0 to 24,489**
- Steps per cycle: **96** (4 × 24 PPQN)
- Encoding: musical beat position relative to loop start
- The counter increments by 96 each beat (at the MIDI-defined resolution)
- At 24,489 steps total: 24,489 / 96 = 255.09... cycles - consistent with bar-length quantization

This information is added to the SP-1-dev GitHub wiki, making the full audio format available for anyone building encoding tools.

**Audio format fully documented** in the GitHub wiki:
- Container: flat binary, no header
- Sector: 8,192 bytes
- Chunk: 2,048 bytes (within sector)
- Samples: 24-bit signed PCM, little-endian
- Channels: 8 interleaved (L/R × 4 stems)
- Timing field: 16-bit counter at chunk start, range 0–24489, 96 steps/cycle
- Sample rate: 48,000 Hz

**maybe's timeline** (post 743) circulates widely - becomes the definitive history of the device. PedalsandChill adds a link to it from the OP.

The bootloader (solderless.engineering) is now stable and several community members have used it to flash custom firmware builds. No reports of bricked units.

**Discussion of encoding tools:** Community members begin writing Python/Rust scripts to convert standard audio files (WAV, FLAC) into the eMMC format. These are shared on GitHub and in the thread.
